# Contributors

Carnelian Core is the result of collaborative effort, innovation, and mentorship from talented individuals who have contributed their expertise, insights, and inspiration to this project.

## Core Team

### Marco Julio Lopes
**Creator & Lead Architect**

Marco is the creator and lead architect of Carnelian Core, responsible for the overall vision, architecture, and implementation of the AI workspace harness. His work spans the Rust core orchestrator, quantum entropy integration, capability-based security model, and the complete agentic execution pipeline.

## Special Acknowledgments

### Justin Oberg
**Early Collaborator & AI Pioneer**

Justin started this journey with Marco by working on getting the original GPT-2 1B model running on local hardware. This early experimentation with local LLM inference kicked off the exploration that eventually led to Carnelian Core's local-first architecture with Ollama integration and GPU support.

### Jonathan Essex
**Cryptography & Database Architecture Mentor**

Founder of [Software Plumbers](https://softwareplumbers.com), Jonathan has been an invaluable mentor throughout Carnelian's development. His deep expertise in cryptography opened Marco's eyes to advanced concepts including:
- Checksum validation strategies and integrity verification
- PostgreSQL transactional ledgers and event sourcing patterns
- Merkle tree architectures for tamper-resistant audit trails
- blake3 hash-chain design for the Carnelian ledger system

Jonathan's insights on modular software design and API architecture directly influenced Carnelian's capability-based security model and the clean separation between the core orchestrator and worker runtimes.

### Vincent Haliburton
**Tokenomics & Web3 Strategy**

Vincent provided inspiration and strategic guidance for Carnelian's planned utility token on Base (Ethereum L2). His expertise in blockchain economics and decentralized systems helped shape the vision for future Web3 integration, including token-gated capability grants and on-chain skill marketplace concepts.

### Marcelino Class
**UI/UX Design & Research**

Marcelino contributed valuable UI/UX suggestions, animation knowledge, and research assistance throughout Carnelian's development. His design insights helped refine the Dioxus desktop UI, including:
- User experience flows for the 17-page desktop interface
- Visual design feedback for the dual-theme brand kit (Forge/Night Lab)
- Animation concepts for real-time event streaming and WebSocket updates
- Research support for agent interaction patterns and workspace automation UX

## Inspirational Sources

### Peter Steinberger
**Creator of OpenClaw**

Peter Steinberger created [OpenClaw](https://github.com/openclaw), an AI agent framework whose pioneering work on tool orchestration and autonomous agent execution provided foundational inspiration for Carnelian's architecture. While Carnelian Core is a fundamentally different implementation built in Rust with a unique security model and quantum entropy integration, OpenClaw demonstrated the potential of AI agent frameworks and influenced early design decisions.

For a detailed comparison of architectural differences, see [LICENSE.md](LICENSE.md#relationship-to-openclaw).

## Community Contributors

We welcome contributions from the community! See [CONTRIBUTING.md](CONTRIBUTING.md) for guidelines on how to contribute to Carnelian Core.

### How to Get Listed

Contributors who submit accepted pull requests will be added to this file with credit for their contributions. We recognize:

- **Code Contributors** — Bug fixes, features, optimizations, and refactoring
- **Documentation Contributors** — Guides, tutorials, API documentation, and examples
- **Testing Contributors** — Test coverage, integration tests, and quality assurance
- **Design Contributors** — UI/UX improvements, branding, and visual assets
- **Community Contributors** — Issue triage, discussions, and community support
